  1. ¿Cuál de las siguientes afirmaciones es correcta sobre los ejecutables y procesos?
    2. Los archivos .JAR son ejecutables y pueden ser ejecutados directamente por el sistema operativo.
    Un proceso es simplemente un archivo que se encuentra en el sistema, independientemente de si está en ejecución.
    Cada instancia del mismo programa en ejecución se ejecuta en un proceso diferente.
    Un servicio es un tipo de ejecutable que requiere la interacción del usuario para funcionar.
  2. ¿Cuál de las siguientes afirmaciones es correcta sobre los estados de un proceso y su planificación?
    3. Un proceso en estado "bloqueado" está en ejecución y no puede ser interrumpido.
    Un proceso pasa de "en ejecución" a "listo" cuando se le acaba el tiempo asignado por el sistema operativo.
    Un proceso en "listo" está esperando la finalización de un evento externo para continuar
    La planificación de procesos permite que un proceso en "bloqueado" ejecute instrucciones de inmediato.
  3. ¿Cuál de las siguientes afirmaciones es correcta sobre la gestión de procesos en sistemas Linux y Windows PowerShell?
    4. El comando ps en Linux solo muestra los procesos en ejecución en primer plano.
    En Linux, un proceso puede ser llevado a primer plano usando el comando bg.
    El comando kill en Linux se utiliza para finalizar procesos utilizando su nombre.
    En Windows PowerShell, se puede ejecutar un comando en segundo plano anteponiendo start-process al comando.
  4. ¿Cuál de las siguientes afirmaciones es correcta sobre la programación concurrente y paralela?
    5. La programación concurrente permite que múltiples procesos se ejecuten simultáneamente en un único procesador
    La programación paralela requiere que las tareas sean ejecutadas en un solo procesador para mejorar el rendimiento.
    La programación concurrente es más simple porque no requiere mecanismos de sincronización.
    En programación distribuida, los equipos clientes son responsables de atender las demandas de otros equipos.
  5. ¿Cuál de las siguientes afirmaciones es correcta sobre las herramientas gráficas para la gestión de procesos en sistemas operativos?
    6. KDE System Guard y Gnome System Monitor son herramientas gráficas exclusivas de Windows
    El Administrador de tareas en Windows solo permite finalizar procesos, sin mostrar información adicional.
    Process Explorer en Windows proporciona información básica sobre los procesos en ejecución.
    Tanto en Linux como en Windows, existen herramientas gráficas que permiten gestionar y visualizar procesos.
  6. ¿Cuál de las siguientes afirmaciones es correcta sobre la programación de hilos en Java?
    7. Para crear un hilo en Java, es necesario implementar la interfaz Runnable y no se puede heredar de la clase Thread.
    El método start() se utiliza para detener un hilo que está en ejecución
    La clase Thread permite que un hilo llame a join() para esperar a que otro hilo finalice.
    El método stop() es recomendado para finalizar hilos de manera segura y eficaz.
  7. ¿Cuál de las siguientes afirmaciones es correcta sobre los estados de un hilo en Java?
    8. Un hilo en estado RUNNABLE está siempre en ejecución y nunca puede ser bloqueado.
    El estado NEW se alcanza cuando se llama al método start() del hilo.
    Un hilo en estado BLOCKED está esperando a que se complete una operación de E/S o intentando bloquear un objeto bloqueado por otro hilo.
    El estado TERMINATED se alcanza únicamente cuando el hilo es interrumpido con el método stop().
  8. ¿Cuál de las siguientes afirmaciones es correcta sobre la gestión de hilos en Java?
    9. El método stop() es la forma recomendada para forzar la finalización de un hilo de manera segura
    Para pausar un hilo, se puede utilizar el método suspend() sin problemas en las versiones más recientes de Java
    Para detener un hilo de forma segura, se debe modificar un atributo booleano que controle su ejecución dentro del método run().
    Se debe usar el método wait() para finalizar un hilo de manera inmediata.
  9. ¿Cuál de las siguientes afirmaciones sobre la sincronización de hilos es correcta?
    10. Los métodos notify() y notifyAll() sirven para que un hilo pause temporalmente su ejecución.
    La palabra reservada synchronized se utiliza para que varios hilos puedan ejecutar un mismo bloque de código simultáneamente
    Los métodos wait(), notify(), y notifyAll() permiten la sincronización de hilos mediante la gestión del acceso a recursos compartidos.
    join() se utiliza para sincronizar los hilos en la ejecución de un mismo bloque de código.
  10. ¿Cuál de las siguientes afirmaciones sobre el modelo Productor-Consumidor es correcta?
    11. El modelo Productor-Consumidor se basa en que los hilos productores consumen datos generados por los hilos consumidores para evitar la pérdida de información.
    En el modelo Productor-Consumidor, los hilos productores y consumidores no pueden coexistir, ya que ambos deben operar en bloques de código independientes.
    El modelo Productor-Consumidor permite la comunicación entre hilos, donde uno o varios hilos producen datos y uno o varios hilos consumen esos datos.
    La técnica de la “píldora de veneno” se utiliza para que los productores detengan a los consumidores cuando han terminado de consumir todos los datos.
